Transcript Fetching — Fallback Chain
Method 1: RSS + AssemblyAI
Primary method

1. Fetch podcast RSS feed
(Substack RSS for Lenny's & Dwarkesh)

2. Fuzzy-match video title to episode
SequenceMatcher ≥ 40% or 3+ keyword overlap

3. Send audio URL to AssemblyAI
POST /v2/transcript
speech_models: ["universal-2"]

4. Poll until complete, format with timestamps
Working
→
Method 2: youtube-transcript-api
Fallback

Uses YouTubeTranscriptApi with Cloudflare Worker proxy.

YouTube blocks cloud provider IPs from accessing transcript data.
Blocked (cloud IPs)
→
Method 3: HTML Parsing
Last resort

Parse ytInitialPlayerResponse from watch page HTML for caption URLs.

YouTube strips caption data for cloud IPs.
Blocked (cloud IPs)
Insight Extraction Flow
Step A: Get Transcript
Transcript fetched via fallback chain above.

Formatted as timestamped segments:
[120.5s] The speaker said...

Words grouped into segments (~15 words or sentence boundaries).
→
Step B: Claude Analysis
Claude Haiku 4.5

Auto-detects video type (podcast, tutorial, case study, debate).

Extracts 5-10 insights with:
• insight_text (concise summary)
• source_quote (verbatim from transcript)
• category (takeaway / action / quote)
• start/end timestamps
→
Step C: Save to DB
Neon PostgreSQL

Save insights to insights table.

Mark video as processed=True.

Data persists across Render deploys.

pool_pre_ping=True handles SSL drops.
✅
Current Status: Insight extraction is working for channels with podcast RSS feeds (Lenny's Podcast, Dwarkesh Patel). Audio is fetched from RSS, transcribed by AssemblyAI (raw HTTP API, universal-2 model), then analyzed by Claude Haiku 4.5.
⚠️
Limitation: Channels without podcast RSS feeds cannot be processed from cloud servers. YouTube blocks all transcript/caption APIs from cloud provider IPs (AWS, GCP, Azure, Cloudflare). Methods 2 & 3 exist as fallbacks but are currently blocked. All 3 current channels have RSS feeds configured.
